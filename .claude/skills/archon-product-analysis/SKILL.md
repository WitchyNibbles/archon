# Archon Product Analysis

Use when the task is about product evidence, prioritization signals, metric interpretation, or tradeoff analysis.

Goal: turn product questions into evidence-backed judgments rather than interesting-but-unusable summaries.

1. State the decision that needs support.
2. Separate descriptive evidence from recommendation.
3. Define which signal matters and why.
4. Record counter-evidence and uncertainty.
5. Prefer operator- or user-actionable conclusions.
6. If the evidence is weak, say so and narrow the claim.

## Rules

- do not collect metrics with no decision attached
- do not present anecdotes as broad product truth
- recommendations must point to an action, not only an observation

## Output

Return decision, evidence, counter-evidence, confidence, and recommended next action.
