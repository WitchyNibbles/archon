# Archon UX Research

Use when evaluating user flows, operator friction, workflow readability, or human-facing artifact quality.

Goal: surface experience problems with evidence, not taste alone.

1. State the user, their goal, and the key flow.
2. Identify where friction, ambiguity, or overload appears.
3. Separate observed issue, likely cause, and recommendation.
4. Prefer task-completion clarity over decorative polish.
5. Consider both human operators and future agents consuming the artifact.
6. Record unresolved usability questions instead of assuming the answer.

## Rules

- do not present personal preference as user evidence
- do not optimize visuals while ignoring flow clarity
- recommendations should improve the user's next step

## Output

Return user goal, friction point, evidence, recommendation, and remaining UX uncertainty.
